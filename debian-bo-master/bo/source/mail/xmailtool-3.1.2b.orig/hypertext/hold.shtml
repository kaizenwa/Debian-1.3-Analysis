Setting the hold Variable

The hold variable is used to indicate that XMailTool should keep messages in the system mailbox unless specificly instructed otherwise. If the hold variable isn't set, XMailTool will move read messages from the system mailbox file to the file pointed to by the MBOX variable.

Examples

set hold
set hold=1
set hold=true
set hold=yes
All of the above are equivalent and indicate that XMailTool should keep messages in the system mailbox unless instructed otherwise. 
set hold=0
set hold=false
set hold=no
All of the above are equivalent and indicate that XMailTool should not keep messages in the system mailbox. XMailTool will copy read messages to the MBOX file.

Default

set hold=0
