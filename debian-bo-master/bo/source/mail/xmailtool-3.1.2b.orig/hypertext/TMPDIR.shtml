Setting the TMPDIR Variable

The TMPDIR variable is used by XMailTool to determine where temporary files should be created. Temporary files are created to hold a copy of the currently opend mailbox file, inbound messages, and outbound messages. Because these files will be accessed much more often than other files, temporary files should be on a local files system rather than one mounted through NFS. Also, temporary files may be very large so the location of the TMPDIR needs to have plenty of disk space.

Examples

csh:

setenv TMPDIR "/usr/tmp"

ksh/sh:

TMPDIR="/usr/tmp"
export TMPDIR

.mailrc or .xmailtoolrc:

set TMPDIR="~/mytmp"
NOTE: This last example shows that TMPDIR can point to any valid directory. It is important to remember that XMailTool performance will be greatly affected by the performace of the file system TMPDIR is on.

Default

"/tmp"
