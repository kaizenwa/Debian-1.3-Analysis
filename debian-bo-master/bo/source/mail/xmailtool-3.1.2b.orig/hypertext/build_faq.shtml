Build Troubleshooting

If you come accross a problem building XMailTool on any platform (not just the ones listed), I would like to hear from you. It is my goal that XMailTool build right out of the box on any Unix platform. Here are some common problems:

Unresolved Externals

If you get an error message indicating that there are unresolved externals, this is because one of the -D options hasn't been specified correctly in either the Imakefile or the Makefile. The links below are the name of the possible unresolved externals and they point to the name of the variable you need to set in the Imakefile and Makefile.
  • setenv
  • truncate
  • mkstemp
  • strstr
  • strerror
  • strcasecmp

More Unresolved Externals

If you get an error message indicating that there were unresolved externals but not one that appears in the list above, then more than likely, one or more of the necessary libraries hasn't been linked in.

socket

If you have any of the following unresolved externals, you probably need something like -lnet or -lsocket on the link command line.
  • socket
  • setsockopt
  • connect

X Libraries

If you get an error message indicating that there were unresolved externals but not one that appears in either of the lists above, and if the name starts with Xt, Xaw, Xmu, X, or has the strings "WidgetClass" or "ObjectClass" in its name, then chances are you are missing at least one of the X11 libraries. Your link line should contain the something similar to one of the next lines:
  • -lXaw -lXmu -lXt -lXext -lX11
  • /lib/libXaw.a /lib/libXmu.a /lib/libXt.a /lib/libXext.a /lib/libX11.a

Warnings

It is unfortunate, but at the current time, XMailTool doesn't build without warnings. In fact, if you are running Solaris, you will see a large number of warning messages that can't be resolved. This is because the compiler complains about every > and < character it sees. If you see any warning messages that cause you concern, please feel free to send me email (bobo@cray.com).

Installation Problems

At some sites, installation in certain directories requires root permission. In any event, I recomend that a system administrator install XMailTool in the "standard location" for your system. If XMailTool isn't installed correctly, the user may get the following error when they try to run XMailTool: 
App-defaults file is not from the same patch level.
Contact your system administrator to install a new copy.

Expecting: [XMailTool v3.1.0], Got: [0]
This is because the app-default file hasn't been installed in the correct location, or the user has specified an XFILESEARCHPATH environment variable that doesn't include the correct location.

Some sites also require that the XMailTool process be in a special mail group when creating lock files, or writing to system mailbox files. If your site requires this, and XMailTool isn't installed with these special premissions, changes made to the users system mailbox will not be efective. At such sites, ls -lg xmailtool would result in the folowing output: 

-rwxr-sr-x   1 root     mail      191872 Oct  2 07:28 xmailtool
