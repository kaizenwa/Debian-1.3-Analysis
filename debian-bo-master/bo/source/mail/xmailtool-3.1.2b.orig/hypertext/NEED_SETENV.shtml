NEED_SETENV

Some machines support a function named setenv(char *var,char *val,int overwrite) which alters or creats an environment variable in the current process space. This function is used by XMailTool to make variables from the .mailrc and .xmailtoolrc files available to child processes such as EDITOR and print_cmd . If your machine doesn't support this function, XMailTool will need the -DNEED_SETENV flag when it is built. This will indicate that it should use it's own version of the function instead of expecting one in libc.

EXAMPLE

     NEED_SETENV = -DNEED_SETENV
XMailTool will use it's own version of setenv(3). 

     NEED_SETENV =
XMailTool will expect to find setenv(3) in libc.
