Setting the saveCurrent Variable

XMailTool can be instructed to return to the last message read in a mailbox file. That is to say that if you are reading message #11, which is from Lisa, in your +work mailbox file. Then you iconify XMailTool or read another mailbox file. Then return to the +work mailbox file, XMailTool will remember that you were reading message #11, which is from Lisa and will display this message first. XMailTool achieves this by adding a header line (X-Current:) to the current message.

The saveCurrent variable is used to indicate that XMailTool should remember the last message you've read from each mailbox file. This variable controls the initial value of the SaveCurrent toggle button.

Examples

set saveCurrent
set saveCurrent=1
set saveCurrent=true
set saveCurrent=yes
All of the above are equivalent and indicate that XMailTool should remember the current message. 
set saveCurrent=0
set saveCurrent=false
set saveCurrent=no
All of the above are equivalent and indicate that XMailTool should not remember the current message.

Default

set saveCurrent=0
