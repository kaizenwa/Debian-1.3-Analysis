Using the ignore and retain commands

The ignore and retain commands are used in the mailrc file to suppress printing of certain header fields when displaying messages. The ignore command is used to specify a list that aren't to be displayed. All other header fields will be displayed. The retain command is used to specify a list of header fields that are to be displayed. All other header fields will be ignored. All header fields will be included when the message is saved unless the alwaysignore variable is set. The format of the ignore and retain commands is as follows: 
ignore keyword1 [keyword2 ...[keywordn]] 
retain keyword1 [keyword2 ...[keywordn]]
keyward1-keywardn are not case sensitive.

Examples

ignore apparently-to default-options errors-to in-reply-to precedence priority
ignore received reply-to return-path return-receipt-to sender status via
retain to from date subject

Note

The initial From line in a message is always printed even if the user specificly instructed XMailTool to ignore it.

Differences

XMailTool handles the ignore and retain commands differently than most mail readers. Each occurance of either the ignore or the retain command causes XMailTool to wipe out its previous ignore/retain lists. The last occurance of the ignore or the retain command will be the one that is used.
