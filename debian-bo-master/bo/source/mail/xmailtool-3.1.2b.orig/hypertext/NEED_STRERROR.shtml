NEED_STRSTR

Some machines support a function named strerror(3) (of the form strerror(erno)) which returns a pointer to the error string associated with that errno value. If your machine doesn't support this function, XMailTool will need the -DNEED_STRERROR flag when it is built. This will indicate that it should use a macro definition of strerror from defs.h instead of expecting one in libc.

EXAMPLE

     NEED_STRERROR = -DNEED_STRERROR
XMailTool will use it's own macro version of strerror(3). 

     NEED_STRERROR =
XMailTool will expect to find strerror(3) in libc.
