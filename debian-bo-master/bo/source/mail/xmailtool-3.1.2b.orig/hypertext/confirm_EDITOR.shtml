Setting the confirm_EDITOR Variable

The confirm_EDITOR variable is used when the EDITOR variable value is something other than "none" to indicate that XMailTool should ask the useruse to confirm delivery of each outbound message when the external editor process has terminated. If confirm_EDITOR is false, XMailTool will use the modification time on the outbound message file to determine if it should send the message. In this situation, if the modification time has changed and the file contains data, XMailTool will pass this file to sendmail. This variable is ignored if EDITOR is set to "none".

Examples

set confirm_EDITOR
set confirm_EDITOR=1
set confirm_EDITOR=true
set confirm_EDITOR=yes
All of the above are equivalent and indicate that XMailTool should ask the user to confirm all outbound messages. 
set confirm_EDITOR=0
set confirm_EDITOR=false
set confirm_EDITOR=no
All of the above are equivalent and indicate that XMailTool should not ask the user to confirm all outbound messages.

Default

set confirm_EDITOR=1
