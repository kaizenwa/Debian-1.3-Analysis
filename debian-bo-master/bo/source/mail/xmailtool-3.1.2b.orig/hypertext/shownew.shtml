Setting the shownew Variable

The first message XMailTool displays after the user has presse the new mail button or de-iconifying the tool window is dependant on a number of user configurable flags. XMailTool first checks to see if saveCurrent has been set and if there is a message with a X-Current: header line. If these two conditions are true, XMailTool will display that message first. If not, XMailTool checks to see if shownew, or showold are set and displays the newest or oldest unread message respectivly. If none of these conditions have been satisfied, XMailTool checks the value of showlast to determine if the first or last message should be displayd. If showlast is set, XMailTool will display the last message in it's list of sorted messages.

Examples

set shownew
set shownew=1
set shownew=true
set shownew=yes
All of the above are equivalent and indicate that XMailTool should display the newest of the unread messages first. 
set shownew=0
set shownew=false
set shownew=no
All of the above are equivalent and indicate that XMailTool should display the newest of the unread messages.

Default

set shownew=0

Note

If both shownew and showold are set, shownew will take priority.
