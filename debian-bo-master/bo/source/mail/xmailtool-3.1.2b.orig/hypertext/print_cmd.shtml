Setting the print_cmd Variable

The print_cmd variable is used to specify the default command used when printing the current message. This string should be a command line that is valid for your value of $SHELL. The current message will be redirected to this processes stdin. It is expected that this command will read its stdin and send its output to the desired printer. Any output from this command will be considerd as an Error message and will be displayed in XMailTools verbose output window. See the Environment Variables XMailTool Sets section to see what variables are set for external processes such as the print_cmd.

Examples

set print_cmd='$HOME/bin/psl | lpr -J "$SUBJECT"'
set print_cmd="mp -p $HOME/mp.pro.ps | lpr -JMail"

Default

The default value of print_cmd is determined from the PRINTER_CMD build flag. Check with the person who built XMailTool to determine what this value is.
