NEED_DIRENT

Some machines support the directory record structure by the name of dirent. These machines will include a header file named dirent.h. Other machines support the directory record structure by the name of direct. These machines will include a header file named sys/dir.h. Some machines support both. If the DIRENT flag is set then the dirent.h file and dirent structure will be used. There are two simple ways of determining how this parameter should be set. If you enter "man dirent" and you get an error, it is likely that you shouldn't use -DNEED_DIRENT. If the file /usr/include/dirent.h exists then you should use -DNEED_DIRENT.

EXAMPLE

     NEED_DIRENT = -DNEED_DIRENT
XMailTool will use struct dirent and #include<dirent.h>. 

     NEED_DIRENT =
XMailTool will use struct direct and #include<sys/dir.h>.
