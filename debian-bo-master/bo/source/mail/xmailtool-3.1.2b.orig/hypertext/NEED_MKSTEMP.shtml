NEED_MKSTEMP

Some machines support a function named mkstemp(3) which is similar to mktemp except that it returns the file descriptor for the newly created temporary file. If your machine doesn't support this function, XMailTool will need the -DNEED_MKSTEMP flag when it is built. This will indicate that it should use it's own version of the function instead of expecting one in libc.

EXAMPLE

     NEED_MKSTEMP = -DNEED_MKSTEMP
XMailTool will use it's own version of mkstemp(3). 

     NEED_MKSTEMP =
XMailTool will expect to find mkstemp(3) in libc.
