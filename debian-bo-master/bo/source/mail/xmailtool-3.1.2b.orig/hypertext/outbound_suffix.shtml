Setting the outbound_suffix Variable

The outbound_suffix variable is used to specify the string used as the suffix of outbound message files. Some editors treat files differently based on their suffix. If you are using such an editor, you may find it very usefull to specify an outbound_suffix value that your editor recognizes.

Note:

Some systems have file name length limits. The length of the file name before applying the outbound_suffix string is 10 ("XMT", 5 decimal digit zero padded process id, 2 hex digit zero padded unique identifier). If your system has a 14 character file name limit, you can specify up to 4 characters including the period (.). Some editors have a limit to the number of charaters it will recognize in a suffix. Check the documentation of your specific editor to determine if any special considerations are necessary.

Examples

set outbound_suffix=".out"
set outbound_suffix=".m"
set outbound_suffix=".outbound"

Default

set outbound_suffix=".mai"
