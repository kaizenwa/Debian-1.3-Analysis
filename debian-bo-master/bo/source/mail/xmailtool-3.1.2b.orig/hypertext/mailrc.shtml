Configuring XMailTool using mailrc

Using the mailrc files is the primary way of configuring XMailTool. There are some differences between the way XMailTool interprets mailrc commands and the way other mail readers interpret mailrc commands. There are also some differences between XMailTool's default variable settings and the default settings for other mail readers. These differences will be discussed in the sections related to those commands and variables.

Mailrc Files

There are three files XMailTool reads at startup and each time the mailrc button is pressed. These files are $MAILRC, $HOME/.xmailtoolrc, and $HOME/.xmailtoolrc.$NAME.

Note:

The value of $MAILRC is originally taken from the environment, but can be changed in the Mailrc files. The current value of $MAILRC is used when the mailrc button is pressed. The value of $NAME is specified with the -name command line option.

Note:

Most mail readers read a sytem mailrc file. XMailTool does not do this as they are normally incomaptable with XMailTool functionality.

Mailrc File Format

Mailrc files are made up of commands and comments. The white space at the begining of a line is ignored. A comment is any line whose first non-white character is "#". Command names and some command options are case sensative. A line may be continued to additional lines by placing a back slash (\) at the end of the line. Lines may not be longer than 1024 characters.

Mailrc File Commands

XMailTool understands 5 commands which are listed below. Lines which contain commands that XMailTool doesn't understand are ignored.
  • alias logical actual1[, actual2[..., actualn]]
  • cd [path]
  • ignore keyword1 [keyword2 ...[keywordn]]
  • retain keyword1 [keyword2 ...[keywordn]]
  • set variable[=['|"]value['|"]]

Mailrc Variables
