KOMA-Script-Paket
=================

Dieses Paket darf nur vollstaendig weitergegeben werden. Aenderungen sind
erlaubt, soweit die geaenderten Dateien mit einem anderen Namen versehen und
die unveraenderten Originaldateien ebenfalls beigelegt werden. Die Weitergabe
geaenderter Dateien ist auch nur bei dtx- und ins-Dateien erlaubt!
Alle Aenderungen sind so zu dokumentieren, dass die Art der Aenderung und der
Urheber der Aenderungen problemlos zu erkennen sind.

Die Dateien duerfen nur unentgeltlich weitergegeben werden. Keinesfalls darf
mit der Weitergabe oder dem Zur-Verfuegung-Stellen ein Gewinn erzielt oder
dieses gewinnorientert durchgefuehrt werden.

Das Paket ist fuer den Gebrauch mit LaTeX2e konzipiert. Ich habe mir groesste
Muehe gegeben, es fehlerfrei und funktionsfaehig zu bekommen. Dennoch
garantiere ich fuer keinerlei Funktion oder Fehlerfreiheit.

Installation:
-------------

* ACHTUNG: Fuer die Installation wird ein komplettes LaTeX2e benoetigt!

  Kopieren Sie zunaechst alle Dateien an einen Ort, an dem TeX/LaTeX sie findet.
  Bearbeiten Sie dann komascr.ins mit LaTeX. Unter UNIX
  z.B. mit:
    tex "&latex" komascr.ins
  oder
    latex komascr.ins
  Sie erhalten eine Reihe neuer cls-, sty- und tex-Dateien. Kopieren Sie diese
  in ein entsprechendes LaTeX-Verzeichnis, damit LaTeX sie findet.

* ACHTUNG: Fuer die Dokumentation wird german.sty (Version 2.5b, notfalls 2.4a)
*          benoetigt!
*          Es ist nicht unbedingt notwendig, die Dokumentation neu zu
*          erstellen, da sie bereits fix und fertig enthalten ist. Sie koennen
*          also den folgenden Abschnitt zunaechst auch ueberspringen. Die
*          Dokumentation dient ausserdem als Beispiel fuer die Verwendung von
*          KOMA-Script.

  Bearbeiten Sie scrguide.tex mit LaTeX. Unter UNIX z.B. mit:
    tex "&latex" scrguide.tex
  oder
    latex scrguide.tex
  Damit das Inhaltsverzeichnis und die Referenzen stimmen, werden insgesamt drei
  Durchlaeufe benoetigt.
  Sie erhalten dabei die Anwender-Dokumentation. Wenn Sie ausserdem einen
  Index erzeugen wollen, so muessen Sie entweder auf beiliegende Datei
  scrguide.ind vertrauen oder vor dem letzen (also dritten) Durchlauf einen
  neuen Index erstellen. Dazu ist die beim vorherigen (also zweiten) Durchlauf
  erzeugte Datei scrguide.idx mit makeindex bzw. makeindx zu bearbeiten.
  Unter UNIX z.B. mit:
    makeindex -g -r -s scrguide.ist scrguide.idx
  bzw.
    makeindx -g -r -s scrguide.ist scrguide.idx

  Wenn Sie ausserdem die Implementierungsdokumentation wuenschen, so bearbeiten
  Sie zusaetzlich die dtx-Dateien mit LaTeX. Unter UNIX z.B. mit:
    tex "&latex" scrclass.dtx
  oder
    latex scrclass.dtx
  In der Regel wird Ihnen die Hauptdokumentation jedoch voellig ausreichen.

+ WICHTIG: Mit scrguide.tex und jeder dtx-Dateien sollten mindestens drei
+          LaTeX-Durchlaeufe durchgefuehrt werden, damit alle Referenzen und
+          Inhaltsverzeichnisse korrekt sind!

  Zum Schluss fuehren Sie bitte noch LaTeX-Durchlaeufe fuer die scr_news-Dateien
  durch. Lesen Sie die gesamte Dokumentation gruendlich durch. Das erspart Ihnen
  viele Fragen.
